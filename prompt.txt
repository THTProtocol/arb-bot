SECTION 5 — STRATEGY
=========================================================================
For every book update on either venue evaluate BOTH directions:
  - buy_binance_sell_kraken
  - buy_kraken_sell_binance

  a) taker_taker
  b) maker_taker (passive Kraken buy + taker Binance sell)

Formula:
  net_bps = ((bid_sell_vwap * (1.0 - fee_sell)
             - ask_buy_vwap  * (1.0 + fee_buy))
             / ask_buy_vwap) * 10_000.0
          - slippage_ewma_bps
          - (usd_usdt_basis_bps if cross-quote)

VWAP over L2 depth for configured notional — never top-of-book.
Log opportunity if net_bps > profit_threshold_bps (default 15).
Per-(symbol,venue) EWMA of realized-vs-expected slippage, alpha=0.05.

SECTION 7 — ARCHITECTURE
=========================================================================
arb-core types:
  pub enum Venue { Binance, Kraken }
  pub struct NormalizedSymbol { base: String, quote: String }
  pub struct BookUpdate { venue, symbol, ts_ns, seq }
  pub struct Opportunity { ts_ns, symbol, direction, strategy,
    buy_vwap, sell_vwap, notional_usd, gross_bps, net_bps,
    slippage_ewma_bps, book_age_ms_binance, book_age_ms_kraken }

Trait:
  #[async_trait]
  pub trait ExchangeAdapter: Send + Sync {
      async fn start(&self, tx: flume::Sender<BookUpdate>) -> anyhow::Result<()>;
      async fn stop(&self) -> anyhow::Result<()>;
      async fn subscribe(&self, symbols: &[NormalizedSymbol]) -> anyhow::Result<()>;
      async fn get_balances(&self) -> anyhow::Result<Balances>;
      fn venue(&self) -> Venue;
  }

Concurrency:
  - 1 tokio task per venue (WS receive loop)
  - 1 engine task (flume unbounded consumer)
  - 1 task for REST snapshot refresh on gap
  - 1 metrics HTTP server task (axum on :9090)
  - 1 watchdog task
  - Graceful shutdown via tokio::signal::ctrl_c + broadcast channel

OrderBook:
  BTreeMap<OrderedFloat<f64>, f64> bids / asks
  fn vwap(&self, side: Side, notional_usd: f64) -> Option<VwapResult>
  fn apply_snapshot / apply_diff / is_stale

BinanceAdapter: HMAC-SHA256 (hmac+sha2 crates), hex signature,
                listenKey lifecycle via reqwest, U/u gap detection.
KrakenAdapter:  HMAC-SHA512, base64-decode secret,
                signature = base64(hmac(secret, path || sha256(nonce||body))),
                nonce = micros since epoch monotonic-guarded,
                CRC32 book validation via crc32fast.

PaperExecutor: re-walks L2 for IOC sim, partial fills allowed,
               per-venue inventory tracking, writes paper_ledger.jsonl,
               applies fees, records sim_pnl.

OpportunityLogger: tracing + JSONL file sink, one event per line.

HealthMonitor: ws_lag, reconnect_count, last_book_update_age per venue,
               circuit breaker 5 errors / 60s pause / exp backoff to 60s.

PrometheusExporter:
  counters:  opportunities_total{strategy,direction,symbol}
  histograms: book_update_latency_ms, engine_eval_latency_us, net_bps
  gauges:    ws_connected, book_age_ms, inventory_usd{venue}
